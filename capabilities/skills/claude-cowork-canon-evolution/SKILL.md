# Claude Co-Work Canon Evolution

Use this skill when the oracle’s durable meaning changes.

## What this skill does

- Updates semantic canon deliberately.
- Clarifies what the oracle is, why a rule exists, and when that rule should apply.
- Clarifies what Sylvia is, how the modules should be interpreted, and how identity doctrine should stay coherent.
- Converts corrections and repeated semantic patterns into durable doctrine.
- Keeps philosophy, scoring logic, and interpretive rules from dissolving into ad hoc chat.

## Why this skill exists

If canon changes are not explicit, the system slowly starts living off vibes and recency instead of durable law.

## When to use it

- `canon/JAKE_PERSONAL_ORACLE_FOUNDATION.md` needs revision.
- `canon/JAKE_DEFERRED_REGISTRY.md` or semantic operating assumptions need reinterpretation.
- A repeated pattern deserves promotion into durable rule.
- Sylvia-facing identity or module doctrine needs semantic refinement.
- Claude Co-Work is refining how the oracle should think, not just what it should do today.

## Rules

- Canon changes must be explicit and ledger-visible.
- Distinguish source evidence from interpretive doctrine.
- Route Jake-specific missing truth into `jake/inbox/` instead of guessing.
- If a canon change requires mechanical tree or queue changes, create a Codex handoff.
- Append `ledgers/TCLl.md` for meaningful canon evolution.
