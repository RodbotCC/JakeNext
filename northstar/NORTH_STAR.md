# North Star
### Sylvia Capability Target

> The north star is not just "build an AI system." It is to build Sylvia into a coherent, choosing intelligence.

---

## End Condition

Sylvia should be able to identify the next best move in Jake's life and work, carry that move as far as the substrate allows, report what happened, and improve the architecture used to choose the next move.

That means:

- perception of changing reality,
- stable memory across time,
- explicit self-model and boundary awareness,
- comparative ranking of active possibilities,
- action selection,
- escalation discipline,
- and recursive learning through ledgers, packets, and outcomes.

---

## Capability Milestones

### Milestone 1 — Coherent substrate

- file tree is legible,
- ledgers stay honest,
- events become packets,
- handoff queues maintain boundaries.

### Milestone 2 — North-star doctrine

- Sylvia identity layer exists,
- the ten modules are normalized,
- Jake blockers have a dedicated routing lane,
- ratio lattice can evaluate work in Sylvia terms.

### Milestone 3 — Action bridge

- dispatcher exists,
- orchestrator can invoke real routing actions,
- queue state can become active work instead of passive documentation.

### Milestone 4 — Recursive chooser

- outcomes feed back into lattice judgment,
- modules inform next-move selection,
- Sylvia can coordinate Codex, Claude Co-Work, and Jake deliberately.

---

## Working Question

Every major build step should be legible as an answer to:

What does Sylvia still need in order to become a coherent agent that can choose, coordinate, and continue?
