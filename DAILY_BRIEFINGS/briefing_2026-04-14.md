### Memory / RAG Alternatives

**1. Observational Memory is the clearest RAG alternative to surface right now.**
Open-source. Uses two background agents to compress conversation history into a dated observation log — no dynamic retrieval. Results: 84.23% on LongMemEval vs. RAG's 80.05% (using GPT-4o), while cutting token costs by up to 10x via prompt caching. The architecture is structurally different from RAG, not just incrementally better — it's worth understanding the design.

**2. Letta (MemGPT) is establishing memory as a first-class agent primitive.**
Memory blocks that agents can explicitly read and write, persisting and evolving identity across sessions. This is the "stateful agent runtime" model — memory as substrate, not retrieval.

**3. Industry consensus is converging on a hybrid architecture.**
Most production agentic systems in 2026 use: factual long-term memory + working short-term context + RAG for corpus lookup. Pure RAG is being repositioned as one tool in a larger memory stack, not the whole answer.

**4. "RAG is dead" is the emerging framing, but the real shift is conceptual.**
The field is moving from "retrieve-then-generate" to "context engine" — intelligent, adaptive, multi-modal context construction. RAG persists as a sub-component; it's no longer the architecture.

### Context Engineering

**1. Context engineering is replacing prompt engineering as the primary discipline.**
The core idea: treat the context window as a scarce resource and engineer everything around it so the model spends attention only on high-signal tokens. This is now being taught and formalized at scale.

**2. ACE (Agentic Context Engineering) framework published.**
Treats contexts as evolving playbooks that accumulate, refine, and organize strategies through generation, reflection, and curation — applied both offline (system prompts) and online (agent memory). +10.6% improvement on agent benchmarks, +8.6% on finance tasks vs. strong baselines.

**3. Context rot is emerging as a production problem.**
As context windows grow with poorly curated information, model performance degrades. Brevity bias and context collapse (where iterative rewriting erodes detail over time) are the two failure modes getting the most attention.

**4. MCP now under Linux Foundation governance.**
The Agentic AI Foundation (formed Dec 2025) now governs MCP, anchored by contributions from Anthropic, OpenAI, and Block. 97M+ monthly SDK downloads. 75+ official connectors. This is infrastructure at this point, not tooling.
