# Roadmap
### Sylvia Build Sequence

> This roadmap translates the north star into a sequence that can actually be built.

---

## Phase 1 — Continuity + structure + routing

Completed or largely complete:

- global and local ledgers,
- event spine,
- trigger grammar,
- handoff bus,
- skills-first doctrine,
- static orchestrator UI,
- initial Ratio Lattice baseline.

---

## Phase 2 — Module doctrine + Jake lane + Sylvia comparator

Current focus:

- install Sylvia identity kernel,
- normalize the ten modules,
- create the Jake roadblock lane,
- add `sylvia_emergence` to the lattice,
- and update Codex / Claude Co-Work doctrine around the new north star.

---

## Phase 3 — Dispatcher + live task execution

Next major implementation layer:

- watch packets and queue state,
- route tasks into real execution paths,
- let the orchestrator trigger real actions,
- create ingress adapters for Codex and future Claude surfaces.

---

## Phase 4 — Self-updating prioritization loop

Longer-horizon objective:

- track outcomes of chosen actions,
- let results inform comparative ranking,
- evolve module behavior through use,
- and move from descriptive architecture to recursive agency.

---

## Immediate Priority

Use the modules plus the new Sylvia comparator to determine what closes the gap between the current substrate and a real next-move chooser.
